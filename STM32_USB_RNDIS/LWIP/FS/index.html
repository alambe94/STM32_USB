 About this demonstration

 

This webserver is a part of a demonstration package developed on the top level of the usb cdc_rndis class.

The Remote Network Driver Interface Specification (RNDIS) Protocol is used to provide a virtual Ethernet link over USB. It is a Microsoft proprietary protocol.
The usbd_cdc_rndis package is a function device implementation of this class. This allows a device to connect to a host system and appear to it as a locally connected virtual Ethernet port.

The demo application handles the Ethernet frames received from the USB endpoitns and hosts a HTTP web server acting as another virtual network device. A standard web browser from host PC can open the web page served from the USB device. The web page provides a statistics of different packets like ICMP, TCP, UDP etc received through the Ethernet frames from the host PC. This demonstrates a simple way in which Ethernet over USB applications can easily be deployed using an STM32-USB device.

 

 

 About Ethernet over USB

 

USB Communications Class is a USB standard that defines mechanism for connecting Networking devices to host machines. USB CDC has specified multiple Subclass standards to support different networking devices. For Ethernet-style networking over USB, one of the following Subclass specifications is used:

1. Remote Network Driver Interface Specification (RNDIS) Protocol:
2. Ethernet Control Model (ECM).
3. Ethernet Emulation Model (EEM).
4. Network Control Model (NCM).

Ethernet over USB provides the following advantages:

. Application independent exchange of data over USB.

. Leverage the networking protocol stack present in operating systems.

. Abstraction for USB application developers at network level like sockets, HTTP (web pages) etc.

. USB-to-Ethernet Adaptors etc.
